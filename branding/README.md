# Keyhop branding assets

Marketing artwork, app screenshots, and source brand references used for
Microsoft Store submissions, GitHub Release banners, and the project
website. **Do not bake these into the binary** — runtime icons live in
`wix/` and `src/windows/`.

## Brand colors (canonical)

The visual identity is driven by what users actually see on screen when
they run keyhop. The canonical palette is taken from the live app:

| Token | Hex | Where it appears |
|---|---|---|
| **Element badge yellow** | `#FFE500` | Hint label background when picking elements (config: `element_badge_bg`) |
| **Window badge blue** | `#33AAFF` | Hint label background in the window picker (config: `window_badge_bg`) |
| **Hint label text** | `#000000` | Foreground letters on every hint badge |
| **Tray icon green-on-yellow** | green letter `K` on yellow square | The `keyhop` system tray icon |
| **Surface dark** | `#0b0e16` to `#1a1d2a` | Settings dialog dark surfaces and marketing backgrounds |

Any new marketing art should lean on **yellow + blue + black** (the
hint colors) rather than introducing new accents.

## Microsoft Store submission map

When uploading to Partner Center → **Store listings → English (United States) → Store listing details**:

| Partner Center field | Asset path |
|---|---|
| Screenshots (≥ 1, recommended ≥ 4) | `screenshots/*.png` (capture more — see "Recommended screenshots" below) |
| Store logos → 1:1 Box art | `store-logo-1080.png` |
| Store logos → 2:3 Poster art | `poster-art-1440x2160.png` |

## Recommended screenshots

Microsoft recommends ≥ 4 screenshots; only `settings-and-tray-menu.png`
is captured today. Capture these on Windows with `Win+Shift+S` (saved
as PNG, landscape ≥ 1366 × 768) and drop them in `screenshots/`:

1. ✅ `settings-and-tray-menu.png` — already captured
2. ⏳ `hint-overlay-desktop.png` — leader chord pressed over File Explorer or another native app, hint labels visible on every clickable element
3. ⏳ `hint-overlay-webpage.png` — leader chord pressed in Edge/Chrome on a content-rich page (the v0.3.0 selling point — web-page hints inside Chromium browsers)
4. ⏳ `window-picker.png` — `Ctrl+Alt+Space` window picker open with several windows visible across one or more monitors
5. ⏳ *(optional)* `hint-overlay-with-arrows.png` — same as #2 with `draw arrow from each badge to its target element` enabled, demonstrating the optional arrow visual

## Regenerating the marketing art

The current `store-logo-1080.png` and `poster-art-1440x2160.png` were
generated by an AI image tool at 1376 × 768 and post-processed to exact
Microsoft Store dimensions (1080 × 1080 square crop + upscale for the
logo; centered 2:3 vertical-strip crop + upscale for the poster).

If you regenerate, keep these constraints:

- **Palette**: matte black keycaps (`#1a1a1f`), white wordmark text
  (`#FFFFFF`), yellow element-hint badges (`#FFE500`) with **black**
  letters, blue window-picker badges (`#33AAFF`) with **black**
  letters, dark-navy background gradient (`#0b0e16` → `#1a1d2a`).
- **Absolutely no green** anywhere in marketing art — green appears
  *only* on the tray icon (the green `K` on yellow square), which is
  intentionally distinct from in-app hint colors.
- After generating, post-process to land on **exact** target
  dimensions. Microsoft Partner Center rejects any logo or poster that
  is off by even a single pixel from `1080×1080` / `2160×2160` (1:1)
  or `720×1080` / `1440×2160` (2:3).
